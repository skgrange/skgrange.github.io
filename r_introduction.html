R resources

Introduction

R is a programming language which is usually described as an environment for statistical computing. Despite R’s domain-specific nature, it has become one of the primary tools for reading, cleaning, tidying, manipulating, exploring, and visualising data. Therefore, R’s abilities extend beyond statistical analyses to most other areas which involve interaction with data. Unlike many other other statistical products, R is free and open-source and has an extremely active community of developers, enthusiasts, and users.

R’s open-source licence has resulted in a huge amount of reproducible R code, most of which is also free and open-source and bound together as downloadable packages which can be used by others. Over 7 000 packages are officially available on the Comprehensive R Archive Network (CRAN) with (probably) thousands more available on GitHub and similar services. This depth and diversity of community development has made R a mature and powerful tool. R continues to increase in popularity, functionality, and scope and can most likely help you if handle data regularly.

History

R has a rather convoluted history and begun life as the S programming language in 1976. S was developed at Bell Laboratories, the same laboratory which produced C, C++, and the Unix operating system. R was first released in 1993 by Ross Ihaka and Robert Gentleman after many years work at the University of Auckland, New Zealand.

After R’s initial release, the functionality and robustness of the language increased, but R remained a rather obscure product. However, more and more users begun to pick-up R when they needed or desired a free statistical package. At some point R ceased to be an exclusive statistical tool and became more utilitarian for data interaction tasks.

R can now read and manipulate almost any data format, has a wide range of data-classes, has powerful packages and functions for data manipulation, and can integrate code with different types of reporting. I predict that in the future, R will be able to service the complete pipeline of data interaction: cleaning to analysis to visualisation to reporting. Interactive, data-driven web content is becoming very popular and R is beginning to integrate all these libraries so their use is easy and convenient. R may also be suitable for tackling the tricky nature of big-data. However, one or two of R’s limitations will need to be overcome, or mitigated for this to occur first.

Technical details

R is primarily a functional programming language. Therefore, programmers apply a series of steps to variables/data-objects using functions. Functions range in complexity and custom functions can be defined and used by an user. For example, perhaps a user wanted to load a data-file containing plant data, check what the table contains, and then plot these data. In R, this could look something like this:

# Load data from a csv file
data_plants <- read.csv("plant_height_data.csv")

# Look at top of table
head(data_plants)

# Plot these data
plot(data_plants)

read.csv, head, and plot are all functions and these are used on the data-object data_plants. This demonstrates the functional way R is designed and used.

R is unconventional in the way that variables are stored as vectors. A vector is simply a series of things stored as a single unit. Because variables are vectors, it is easy to do things to the entire vector without constructing for-loops. For example, adding three to a number is done like so:

# Define
x <- 5

# Add three to x
x + 3
## [1] 8

but adding three to a vector of numbers is done in the same way (c() is used to combine/concatenate elements):

# Define
y <- c(5, 6, 7, 8, 9, 10)

# Add three to x
y + 3
## [1]  8  9 10 11 12 13

For those which are unfamiliar with programming, this may seem like a minor point. However, very few languages operate in this way and it allows for clear, readable code, especially when the complication increases. The addition of numbers as above is a trivial example, but this behaviour is preserved across R and it allows for the use of functions in complicated scenarios without the need to use large amounts of “book-keeping” code.

R’s combination of a large package library, its vectorisation, and the ease of function development, has made the language capable at doing things quickly and correctly. In my opinion, this is what makes R so powerful, it has captured the essence of doing something, and then moving on. This being said, R is not a toy-language and it is also suitable for building large and scalable programmes.
